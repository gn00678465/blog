物件參考觀念-延伸範例

複雜的情況 1

宣告變數 family 並設立成Object時，會在記憶體中給它一個位址（0x01），於 family 內有另一個物件 merbers，這時候，一樣會在記憶體中並給它一個位址（0x02）。

            graph LR
            A[family:0x01] -->|0x01| B(name: '小明家'
merbers: 0x02)
B -->|0x02| C(father: '老爸'
mon: '老媽'
ming: '小明')
style B fill:#ccf,stroke:#f66,stroke-width:2px,stroke-dasharray: 5, 5
style C fill:#ccf,stroke:#f66,stroke-width:2px,stroke-dasharray: 5, 5

定義一個變數 merbers 並指向 family.merbers 的參考物件(0x01)。

            graph LR
            A[family: 0x01
merbers: 0x02] -->|0x01| B(name: '小明家'
merbers: 0x02)
B -->|0x02| C(father: '老爸'
mon: '老媽'
ming: '小明')
A --> C
style B fill:#ccf,stroke:#f66,stroke-width:2px,stroke-dasharray: 5, 5
style C fill:#ccf,stroke:#f66,stroke-width:2px,stroke-dasharray: 5, 5
  • 觀念 1
  • 觀念 2

觀念 1.

將 merbers 賦予一個新物件 {ming: '大明'} ，這時候，記憶體中會給它一個新位址（0x03），並指向此位址。

使用 {} 賦予一個新物件，一律會產生新的 參考物件。

            graph LR
            A[family: 0x01
merbers: 0x03] -->|0x01| B(name: '小明家'
merbers: 0x02)
B -->|0x02| C(father: '老爸'
mon: '老媽'
ming: '小明')
A -->|0x03| D(ming: '大明')
style B fill:#ccf,stroke:#f66,stroke-width:2px,stroke-dasharray: 5, 5
style C fill:#ccf,stroke:#f66,stroke-width:2px,stroke-dasharray: 5, 5
style D fill:#ccf,stroke:#f66,stroke-width:2px,stroke-dasharray: 5, 5

此時所有物件皆為獨立，並無指向相同位址。

觀念 2.

如只單單修改屬性的值，未使用 {} 賦予一個新物件的情況下，此時仍指向相同的參考物件。

複雜的情況 2

step 1

  •             graph LR
            A[a: 0x01
    b: 0x01] -->|0x01| B( x: 1 )
style B fill:#ccf,stroke:#f66,stroke-width:2px,stroke-dasharray: 5, 5

step 2

  • a = { x: 2 } 是一個賦值運算式，且有新增一個物件，故記憶體中會給它一個新位址（0x02），並指向此位址。 
                graph LR
            A[a: 0x02
    b: 0x01] -->|0x01| B( x: 1 )
A -->|0x02| C( X: 2 )
style B fill:#ccf,stroke:#f66,stroke-width:2px,stroke-dasharray: 5, 5
style C fill:#ccf,stroke:#f66,stroke-width:2px,stroke-dasharray: 5, 5
  • 因 a.y 指向原本的參考路徑
                graph LR
            A[a: 0x02
    b: 0x01] -->|0x01| B( x: 1
    y: 0x02 )
A -->|0x02| C( x: 2 )
B -->|0x02| C
style B fill:#ccf,stroke:#f66,stroke-width:2px,stroke-dasharray: 5, 5
style C fill:#ccf,stroke:#f66,stroke-width:2px,stroke-dasharray: 5, 5
  • 目前 a 指向（0x02），0x02內 y 不存在，故回傳 undefined
    b 指向（0x01），故回傳 {X: 1, y: {x: 2}}
